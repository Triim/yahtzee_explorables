# Промпт для Claude Code — переделка Yahtzee Explorable

Ты дорабатываешь интерактивный лонгрид-«объяснялку» про игру Yahtzee. В репозитории **уже есть сборка со СТАРЫМ контентом** (он переведён, но устарел) и со сломанной интерактивной механикой. Твоя задача — **заменить контент на новый чистовик и перестроить взаимодействие по правилам ниже**. Двигайся строго по порядку, по одному разделу за раз, и после каждого проверяй приёмку. Не выдумывай структуру и числа — бери их из файлов.

## 0. Главное, что нужно понять про переделку

- Прежняя механика была сломана так: читатель просто скроллил текст мимо статичной модели, взаимодействия не было, модели несли свои подписи и дублировали текст, анимация броска дёрганая. **Всё это переделать по правилам в разделах 2–4 этого промпта.**

## 2. Единица контента — БИТ (это ядро переделки)

Каждый раздел состоит из **битов**. Один бит = один экран и устроен так:

1. **Левый текст (prompt):** одна фраза/вопрос/мысль появляется слева.
2. **Обязательное действие на модели (гейт):** читатель бросает / тянет / выбирает / держит. **Пока действие не сделано — скролл заблокирован, дальше не пройти.**
3. **Ответ модели + самоменяющийся текст (payoff):** модель видимо реагирует, и слева сам дописывается ответ/определение. Появляется стрелка «дальше».
4. Биты без модели (помечены «без модели») — чисто текстовые, для ритма; их пропускают сразу.

Правила:
- **Модель отвечает на текст и не несёт своих повествовательных подписей.** Никаких «A single die. Roll it.» в правой панели — только функциональные микроподписи (оси, кнопки, числа). Весь рассказ — слева.
- **Гейтинг — через блокировку прохода, без перехвата колеса.** Нативный скролл; следующий бит/стрелка открываются только после взаимодействия. Не делать scroll-hijacking (не менять скорость, не трапить колесо).
- **Варьируй механику** от бита к биту (бросок → тяни ползунок → выбери → собери), не повторяй один приём подряд.
- Состояние модели — чистая функция активного бита (скролл вверх обратимо убирает разблокировки).

## 3. Глобальная оболочка (из `…section-00-intro.md`)

- **Hero на весь экран:** заголовок «Пять кубиков» + короткое описание + стрелка «вниз». На самом hero моделей нет.
- **Меню навигации:** сквозное, со списком Введение + 10 разделов, текущий подсвечен, по клику — переход к началу раздела.
- **Хореография въезда:** при входе в раздел сначала въезжает **левая колонка с текстом**, на следующий скролл — **правая модель**. Не обе сразу.
- **Самоменяющийся текст:** левый текст дописывается/меняется в ответ на действие (в Разделе 1 это уже работает — взять за образец).

## 4. Анимация броска (заменить дёрганую)

Бросок должен ощущаться как бросок: замах → дуга с реальной подкруткой (≈540°) и замедлением (ease-out) → приземление с отскоком; тень сжимается на подлёте и расплющивается при ударе. Грани быстро мелькают и садятся на честный результат движка. Несколько кубиков — со сдвигом ~70 мс. `prefers-reduced-motion` → мгновенно, без движения. Никакого `infinite`-тряса.

## 5. Технические рамки (не нарушать)

- **Vite + React + TypeScript.** Engine — чистый TS, без React/DOM (`src/engine/`).
- **Только hand-rolled SVG** для всех визуалов (кубики, гистограммы, сетка 6×6, граф состояний, воронка). Без chart-библиотек.
- **KaTeX** для формул.
- **Состояние в памяти** (React state). Без localStorage/sessionStorage.
- **DP считается офлайн** Node-скриптом → шиппится как таблица-«оракул»; в браузере только смотрим значения, не решаем DP на лету. **Монте-Карло — в web-worker**, чтобы скролл не лагал.
- **Честность интерактива:** настоящий бросок всегда настоящий; постановочные показы (максимум 1575) явно помечены как демонстрация, без кнопки-имитации броска.
- **DP, оракул, воркер — это реализация за кадром, читателю НЕ называются.** Раздел 7 называется «Ценность положения», а не «динамическое программирование».
- Desktop-first; на узком экране — заглушка «открой на компьютере».

## 7. Порядок работы и приёмка

Делай по одному за раз, коммить после каждого:

1. **Сначала оболочка:** hero, меню, двухколоночный каркас, хореография въезда, гейтинг скролла, самоменяющийся текст, нормальная аним组ация броска. Приёмка: hero → на скролл въезжает левый текст, затем правая модель; до взаимодействия дальше не пускает; после — пускает.
2. **Затем разделы по очереди** (1→10): для каждого вставь биты из его файла (левый текст, модель из *(модель: …)*, гейт, самоменяющийся текст). Старый контент этого раздела удали.

**Приёмка каждого раздела (все пункты):**
- Старого текста/шагов не осталось; текст совпадает с файлом чистовика.
- Каждый бит читается как «фраза → действие → ответ»; до взаимодействия скролл заблокирован.
- Модель отвечает на текст и не несёт повествовательных подписей.
- Модель усложняется по ходу раздела, а не показана сразу целиком.
- Формулы рендерятся через KaTeX там, где они есть в тексте.
- Настоящие броски настоящие; показы (1575) помечены как демонстрация.

После каждого раздела покажи короткий отчёт, как выполнена приёмка, и переходи к следующему.
